<Cmd name="form" /> 讓你設計自己的表單。成員在面板上按一顆按鈕、填完送出，機器龍就開一個私人討論串，把填表的人和管理員一起拉進去，後續直接在裡面談。

檢舉表單、進服申請、客服單，都是同一套流程。整趟長這樣：

## 建立表單

<SlashCommand name="form create" description="跳出表單，填標題和通知對象" />

送出指令後填這幾個欄位：

| 欄位 | 說明 |
| ---- | ---- |
| 標題 | 最多 45 字，也是成員填表視窗的標題 |
| 說明 | 選填，會顯示在面板訊息上 |
| 按鈕文字 | 選填，不填就叫「填寫」 |
| 通知身分組 | 1 到 5 個，有人送出表單時會標註他們 |

建好之後會出現一張只有你看得到的管理卡。加題目、發佈面板、停用，全部在這張卡上操作，卡片會原地更新，不會洗版：

## 加題目

在管理卡上按「新增問題」，一次加一題，最多 5 題（Discord 表單視窗的欄位上限）。

題目分三種：

| 類型 | 說明 |
| ---- | ---- |
| 短答 | 一行文字 |
| 段落 | 多行文字，適合「請描述發生什麼事」這種題目 |
| 單選 | 下拉選單，選項填 2 到 25 個，用逗號或換行分隔 |

發佈之後照樣能加題、刪題，不用重發面板。成員每次點按鈕，機器龍都會拿最新的題目組表單。

## 發佈面板

在管理卡上按「發佈面板」，選一個頻道，機器龍就把面板發過去。

## 成員送出之後

面板頻道底下會多一個私人討論串，名字像 `#0042 阿貓`，編號照送出順序排，掃一眼討論串列表就知道有幾張單。

機器龍會在討論串裡標註填表的人和通知身分組，答案整理成一張卡。頻道裡的其他人看不到這個討論串。

談完之後按「結案」，討論串會封存並上鎖。只有通知身分組的成員、或有「管理討論串」權限的人按得動。

## 管理表單

<SlashCommand name="form list" description="列出這個伺服器的表單，選一張開管理卡" />

每個伺服器最多 10 張表單。停用之後面板按鈕還在，只是成員點了會被告知表單已停用；再按一次「啟用」就恢復。
